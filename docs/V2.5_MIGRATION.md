# ETF Engine v2.5 migration

## Final locations

- `src/etf_engine/providers/official_openapi.py`: TWSE/TPEx OpenAPI catalog discovery and normalization
- `src/etf_engine/providers/holdings.py`: holdings provider adapters
- `src/etf_engine/services/sync_tw_entities.py`: rebuild Taiwan entity universe
- `src/etf_engine/services/sync_holdings.py`: sync TW/US holdings with cache preservation
- `src/etf_engine/services/build_classifications.py`: generate taxonomy
- `src/etf_engine/services/public_builder.py`: publish v2.5 JSON
- `data/seed/entities_us.json`: maintained US universe
- `data/seed/tw_entity_overrides.json`: Taiwan aliases/overrides
- `data/seed/classifications_curated_us_ai.json`: curated AI tags
- `data/seed/holdings_manual/`: optional issuer/manual holdings JSON or CSV
- `scripts/rebuild_all.sh`: complete rebuild

## Manual holdings format

`data/seed/holdings_manual/TW-0050.csv`:

```csv
holding_symbol,holding_name,weight,as_of
2330,台積電,58.0,2026-07-18
```

Weights may be `58`, `58%`, or `0.58`.

## Commands

```bash
pip install -e ".[dev]"
python -m etf_engine.cli validate
pytest -q
bash scripts/rebuild_all.sh
```

The entity sync intentionally fails if `0050` is missing or fewer than 200 Taiwan ETFs are collected. A failed holdings refresh preserves the last successful cache.
